补题

Posted on 2021-02-07 Edited on 2021-02-18 In acm

POJ用来搞自己心态
HDU用来做水题
CF用来训练思维

K Best
显然贪心不可解，可以考虑把分母移到左边然后二分S(s)，每次二分的时候排个序，最后输出前k个值的序号即可
How Many Tables
题目大意就是求并查集的类型的个数，如果了解并查集的实现原理的话可以发现对于所有par[i]=i的都是根结点，所以只需要遍历1到n每次遇到这样的结点ans++即可
小希的迷宫
其实就是给一堆结点，每次连边的时候用并查集unite一下，如果两个结点属于同一个连通块，说明最终结果不是一棵树，用flag标记一下即可，另外如果最终情况下整个图不是连通的，那也不是一棵树，可以每次用set存结点，每次unite的时候边数+1，如果边数=结点数-1的话，说明是一棵树，这题还有个坑点就是单次输入0 0的时候要特判为Yes
Is It A Tree?
和上一题不一样，这里是有向图，所以每次连边的时候需要标记一下被指向的结点，如果某个结点被指向了两次以上，flag=false，其余和上一题一致
More is better
并查集，需要找出最大的集合的大小，修改一下rk数组，一开始每个集合初始化为1，然后每次合并的时候加上另一个集合的大小，最后找出rk数组中最大的值即可，注意这里每次给的两个数可能会有重复，要提前find一下根结点防止多加rk
Constructing Roads
最小生成树，第一次输入给了一个邻接矩阵，把他转成edges里即可，第二次输入相当于在kruskal算法前先合并一部分的结点，然后再跑一遍kruskal即可，因为先前已经合并了一部分集合所以在kruskal中会跳过那些结点
畅通工程
并查集，找出没有被合并的结点，跑一遍par然后找出par[i]=i的情况数，然后-1即可
还是畅通工程
最小生成树板子题，没什么好说的，直接套板子即可
畅通工程2
最小生成树+并查集，先跑一遍kruskal然后跑一遍par找有没有未被合并的结点
畅通工程再续
因为给的是坐标，所以自己写一个point类+dis函数，然后跑N*(N-1)/2次把每个边加到edges中，跑kruskal每次合并的时候加个>=10和<=1000的条件即可
继续畅通工程
和前面的题差不多，先把一部分合并，然后跑一遍最小生成树
Connect the Cities
模板题，秒了
Eddy’s picture
类似畅通工程再续那题，把边界条件去掉即可
Jungle Roads
把数字编号换成字母即可
Farm Irrigation
我做的有点麻烦，每个节点多开了上下左右四个节点，第一次循环按照要求对每个结点的上下左右赋值，第二波对每个结点的上下左右找符合条件的结点，所以写了两个switch，如果找到则unite一下，最后数一下集合的个数就可以了，因为输入的是一个类似矩阵的玩意儿，把二维数组展开成一维的就可以了
A Bug’s Life
种类并查集，开2*N空间的数组，前N个代表bug自身性别，后N个代表bug的对立性别，例如1和2可以互动，则代表1和2的对立属于一个性别，同理1的对立和2属于一个性别，如果两个bug属于同一个集合但是可以互动，说明存在suspicious bug
Virtual Friends
因为输入是string，一开始想用map做并查集，后来发现这好像不太现实，实际上每个case开个map记录每个名字依次出现的顺序就可以了，然后按照常规的并查集做
Zjnu Stadium
带权并查集，每次在路径压缩的时候顺带加上其父节点到根结点的距离就可以了，这里要注意先递归压缩所有父节点的路径，然后再更新dis，否则只会更新一次，两个集合合并的时候b集合的根结点指向a集合的根结点，然后更新b集合根结点到a集合根结点的距离，注意如果存在两个集合可以进行合并的话(即!same(a,b))，必然意味着两个集合的根结点的dis均为0，所以这里不需要做一些运算
Building Block
带权并查集，这题中把a集合合并到b集合只需要更新a的dis的值为b集合的大小即可，因为是原题的意思是叠在上方，所以多开一个数组记录集合的大小即可，做完这题也可以发现带权并查集关键还是掌握思想，并不存在万能的模板
How far away ?
乍一看以为是最短路问题，题干中有一条很关键的信息，就是无环，即整个结构是一颗树，面对如此庞大的查询，可以想到用LCA可以较快地得出最短路，比如a和b的最短路，就是a和b到最近公共祖先c的距离然后相加，LCA是很好求的，但是问题在于如何求出a和b到LCA的距离，这里可以在DFS的过程记录根结点到某结点的距离，这样的话a和b的距离就是$dis[a]+dis[b]-2*dis[LCA(a,b)]$，然后就AC了
Connections between cities
和上一题差不多的思路，只不过这不是树而是森林，所以用并查集先合并结点，然后去DFS并查集中的根结点，最后LCA解决即可
HDU Today
map+dijkstra，注意起点和终点一样(我怎么老是想不到这种骚数据)
Arbitrage
数据量很小，所以floyed跑一遍然后找g[i][i]中是否存在比1大的值就可以了，递推式$g[i][j]=max(g[i][j],g[i][k]*g[k][j])$
Add and Divide
估算一下可以发现b越大，a/b的次数的递减速度是很慢的，这样的话增加b的成本就比a/b的成本大了，所以可以枚举b到一个比较小的数(可以算一下最坏情况，即$a=10^9,b=1$，可以发现最多操作30次)，然后依次算操作次数即可(满脑子在想$O(1)$的算法，我是sb)
Idiomatic Phrases Game
按序号存idiom即可，然后按照题意模拟一下最后dijkstra即可，这里存string还是挺搞的，RE和MLE了好几次
Choose the best route
对于这种多个起点一个终点的问题来说，显然多次dijkstra会超时，可以反向建图，从终点跑dijkstra然后遍历起点找最小值即可
In Action
dijkstra+背包，把占用功率和最短路分别看做背包容量和物品价值，设总功率为sum，遍历sum/2+1到sum找最小值即可
六度分离
乍一看是并查集，实际上是求最短路长度，用的dijkstra然后遍历一遍找是否存在长度大于7的即可
find the safest road
Floyed可以解，改一下递推方程即可
find the mincost route
无向图Floyed求最小环问题，关键在于理解floyed算法的本质，floyed的最外层循环实际上是只用前K个结点作为中转算出i到j的最短路，那么在遍历到k时，已经求出了前k-1个结点对应的最短路，也就是说对于所有小于k的i,j的最短路都已经求得了，最小环就是i到j的最短路+i到k到j的路径，注意后者不是最短路，是原图的路径，因为题目要求不能重复经过结点，然后遍历一下即可
Bus System
看完题我居然觉得这题是背包，然后感觉好像挺麻烦的，其实还是最短路，用cost作边权，一开始的边权就是两点距离对应的cost，然后用Floyed更新，注意INF开的大一点，因为题目给的数据很大
Max Sum
一开始右指针不断向右跑，并不断更新max_sum，如果发现当前的累加值小于0，代表可以重新找一段新的max_sum，注意这个时候保存的是前面一段的max_sum，如果重新找的值比max_sum大则更新，并更新左指针，这样$O(n)$就可以结束了
Cable master
二分搜索，C(x)代表以x作为长度能否切割出大于等于k条绳子
Aggressive cows
贪心+二分，先对所有的x坐标排个序，C(x)代表能否将k头牛以x为最小距离分配牛舍，分配的时候从第一个位置开始以贪心的思想分配即可
River Hopscotch
对于整数二分来说，主要看check函数，我一般将check函数用来开区间缩减上界，因为最后的lo和hi相差一定不超过1，既然上界是开区间，那么只能是下界，如果题目要求是double类型的话，mid就行，因为可以把整个函数看做连续的
